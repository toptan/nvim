## ADDED Requirements

### Requirement: mini.clue popup on Leader groups

The system SHALL show a key-clue popup, via `mini.clue` (part of `mini.nvim`, configured in
`plugin/60-mini.lua`), when the user presses `<Leader>` in normal or visual mode and pauses. The
popup's group descriptions SHALL be sourced directly from `Config.leader_group_clues`
(`plugin/20-keymaps.lua`), not a separate or duplicated table, so the two never drift out of sync.

#### Scenario: Pausing after Leader in normal mode

- **WHEN** the user presses `<Leader>` in normal mode and waits
- **THEN** a popup appears listing each configured `<Leader>` prefix group and its description
  from `Config.leader_group_clues` (e.g. `b` -> "+Buffer")

#### Scenario: Pausing after Leader in visual mode

- **WHEN** the user presses `<Leader>` in visual mode and waits
- **THEN** a popup appears listing the visual-mode entries from `Config.leader_group_clues`

#### Scenario: Completing a full sequence

- **WHEN** the user presses `<Leader>` followed (with or without pausing to view the popup) by the
  remaining keys of an existing leader keymap (e.g. `<Leader>tf`)
- **THEN** that keymap's command runs exactly as it would without `mini.clue` configured

### Requirement: Built-in trigger groups (g, z, marks, registers, windows, insert/cmdline completion)

The system SHALL show a `mini.clue` popup for the following additional trigger prefixes, using
`mini.clue`'s own built-in `gen_clues` generators for their descriptions (no hand-authored clue
text): `g` (normal, visual), `z` (normal, visual), `` ` `` and `'` (marks; normal, visual), `"`
(registers; normal, visual), `<C-w>` (windows; normal), and `<C-r>` (registers; insert, cmdline).

#### Scenario: Pausing after g

- **WHEN** the user presses `g` in normal mode and waits
- **THEN** a popup appears listing `g`-prefixed commands and their descriptions, as generated by
  `MiniClue.gen_clues.g()`

#### Scenario: Pausing after a register or mark prefix

- **WHEN** the user presses `"` (or, in normal mode, `'`/`` ` ``) and waits
- **THEN** a popup appears listing the available registers (or marks), as generated by
  `MiniClue.gen_clues.registers()` (or `.marks()`)

#### Scenario: Pausing after Ctrl-W

- **WHEN** the user presses `<C-w>` in normal mode and waits
- **THEN** a popup appears listing window commands, as generated by `MiniClue.gen_clues.windows()`

#### Scenario: Pausing after Ctrl-R in insert or command-line mode

- **WHEN** the user presses `<C-r>` in insert mode or command-line mode and waits
- **THEN** a popup appears listing the available registers for insertion, as generated by
  `MiniClue.gen_clues.registers()` / `.builtin_completion()`

### Requirement: mini.clue popup on bracket-navigation groups

The system SHALL show a `mini.clue` popup for the `[` and `]` prefixes (normal and visual mode)
used by `mini.bracketed` (see the `bracket-navigation` capability). No separate clue descriptions
SHALL be authored for these groups; the popup SHALL rely on the `desc` already set by
`mini.bracketed`'s own mappings.

#### Scenario: Pausing after an open-bracket prefix

- **WHEN** the user presses `[` (or `]`) in normal or visual mode and waits
- **THEN** a popup appears listing `mini.bracketed`'s available suffixes for that prefix, using
  each mapping's own description

### Requirement: No triggers beyond the configured set

The system SHALL NOT configure `mini.clue` triggers for prefixes other than the ones listed above
(`<Leader>`, `g`, `z`, marks, registers, windows, and bracket-navigation) as part of this
capability.

#### Scenario: Pressing an unconfigured prefix key

- **WHEN** the user presses a prefix key that is not one of the configured triggers
- **THEN** no `mini.clue` popup appears, and the key behaves exactly as it would with `mini.clue`
  not installed
