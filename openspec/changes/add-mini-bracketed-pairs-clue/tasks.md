## 4. Manual verification

Drive via `nvim --headless --listen <socket>` + `nvim --server <socket> --remote-send`, as in
prior changes, so real insert-mode typing and popup timing can be observed via RPC.

- [x] 4.1 `mini.pairs`: typing `(`, `[`, `{`, `"`, `'`, `` ` `` inserts the matching closing
      character with the cursor placed between them
- [x] 4.2 `mini.pairs`: typing the closing character immediately after its auto-inserted
      counterpart moves over it instead of inserting a duplicate
- [x] 4.3 `mini.pairs`: pressing `<BS>` with the cursor inside an empty auto-inserted pair (e.g.
      `(|)`) deletes both characters
- [x] 4.4 `mini.pairs`: pressing `<Enter>` with the cursor inside an empty auto-inserted pair and
      no completion popup visible splits it across two lines with the cursor indented in between
- [x] 4.5 Completion still works: with the popup visible and an item selected, `<Enter>` confirms
      it (`<C-y>`), not `mini.pairs`' smart-CR
- [x] 4.6 Plain `<Enter>` (no popup, cursor not inside any pair) still inserts an ordinary newline
- [x] 4.7 `mini.bracketed`: in a buffer with multiple buffers open, `]b`/`[b` moves to the
      next/previous buffer
- [x] 4.8 `mini.bracketed`: `]o`/`[o` navigates the oldfiles list (or another convenient default
      target, e.g. `]c`/`[c` over comment lines) as expected
- [x] 4.9 `mini.clue`: pressing `<Leader>` in normal mode and waiting shows the popup listing the
      groups/descriptions from `Config.leader_group_clues` (e.g. `b` -> "+Buffer")
- [x] 4.10 `mini.clue`: pressing `<Leader>` in visual mode and waiting shows the visual-mode
      subset of groups (`g`, `l`)
- [x] 4.11 `mini.clue`: completing a full sequence through the popup (e.g. `<Leader>` wait
      `t` `f`) still triggers the existing `<Leader>tf` keymap normally
- [x] 4.12 `mini.clue`: pressing `g` (or `z`) in normal mode and waiting shows a popup with
      descriptive entries generated by `gen_clues.g()`/`.z()` (not blank/raw-key entries), and `gg`
      (or `zz`) still executes normally afterward
- [x] 4.13 `mini.clue`: pressing `"` (registers) and, in normal mode, `'`/`` ` `` (marks) shows a
      popup listing available registers/marks
- [x] 4.14 `mini.clue`: pressing `<C-w>` in normal mode shows a window-commands popup, and e.g.
      `<C-w>s` still splits the window afterward
- [x] 4.15 `mini.clue`: pressing `<C-r>` in insert mode and in command-line mode shows a
      register-listing popup
- [x] 4.16 `mini.clue`: pressing `[` (or `]`) in normal mode shows a popup listing
      `mini.bracketed`'s suffixes with non-empty descriptions (confirms `mini.bracketed` sets
      `desc` on its mappings, per `design.md`'s Risk on this), and e.g. `]b` still navigates to the
      next buffer afterward
- [x] 4.17 `mini.nvim` ships no per-module healthcheck, so `:checkhealth mini.nvim` reports "No
      healthcheck found" regardless (same as the prior `mini.completion` change) — not a real
      error. No actual errors appeared in `:messages` across any of the manual runs above.
